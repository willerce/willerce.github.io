浅谈Discuz! Support Team

2008年09月08日

或许，在谈Discuz! Support Team(以下简称DST)之前，我应该先介绍一下Discuz!这个基于PHP+Mysql的程序。可能绝大部分网民更了解Wordpress这个同是PHP+Mysql的博客程序，但如果你访问国内的Wordpress中文论坛，或者Wordpress爱好者，Wopus论坛，你可能会注意到论坛Footer里的声明：Powered by Discuz!
可以说，Discuz!在国内占绝对领先地步。

康盛创想对Discuz! 的概述：

Crossday Discuz! Board 论坛系统（简称 Discuz! 论坛，中国国家版权局著作权登记号 2006SR11895）是一个采用 PHP 和 MySQL 等其他多种数据库构建的高效论坛解决方案。作为商业软件产品， Discuz! 在代码质量，运行效率，负载能力，安全等级，功能可操控性和权限严密性等方面都在广大用户中有良好的口碑。凭借 Discuz! 开发组长期积累的丰富的 web 开发及数据库经验，和强于创新，追求完美的设计理念，使得 Discuz! 在很短时间内以其鲜明的个性特色从国内外同类产品中脱颖而出。经过了效率最优化和负载能力最佳化设计的 Discuz! ，已获得业内越来越多专家和权威企业的认可。

对于站长而言，利用 Discuz! 均能够在最短的时间内，花费最低的费用，采用最少的人力，架设一个性能优异、功能全面、安全稳定的社区论坛平台。

对于网民而言，注册任何一个由 Discuz! 软件系统建立的网站/论坛，将能方便、快捷地享受到论坛带来的互动体验，进行发/回帖、添加/修改资料、站内短信、社区搜索等数百项基本论坛操作，以及进行 Wap 访问、社区交易、论坛悬赏等数十项高级社区应用。
那么Discuz! Support Team是什么呢？DST成立于2006年12月25日，DST的大大伊泽浩在Discuz! Support Team 即将宣告成立一帖中这样说：

Discuz!支持团队（Discuz! Support Team）是具有一定PHP能力的使用Discuz!的PHPers或Discuz!爱好者组成的团队。

她属于支持Discuz!系列程序的受Discuz!官方支持的独立第三方团队，面向PHPers和Discuz!爱好者、使用者，服务于PHPers和Discuz!爱好者、使用者。

她将与Comsenz合作，根据用户的需求和团队成员制作/修改/更新大量插件、Hack、风格和相关程序、文档（包括服务器协助配制软件、转换程式、新手教程、插件/HACK字典等）。

她和她的每个成员都以“我为人人，人人为我”为原则，积极奉献，立志创造良好的讨论氛围、引导新手。以爱好促友谊，以虚拟网络为手段培养真实感情。
自从DST于2006年12月25日正式成立以来，便集合了多方的热心朋友与人才，时时刻刻都默默地为广大的Discuz!用户和支持者们提供着各种类型的免费服务。

DST所提供的服务范围非常之广泛，从风格和插件的开发及升级，疑难解答，新手教学，到发布资料片类的整合型风格等等，让许许多多使用Discuz!的站长和用户们受益良多。
DST这一路走来，历经艰辛，面对挑战，依靠团队精神坚持到今天，真的是很不容易啊。

我是DST主题组的成员之一，我在2007年7月29日的时候开始加入DST，当时我使用Discuz!已经有一段的时间，并对Discuz主题有些了解，带着学习与分享的目的加入了DST。当时是向DST的创始人伊泽浩申请的，伊泽浩是个很好的人，可能这个好太广泛的，但他给我的就是这样的一个感觉。我问他我加入DST需要做些什么的？他是这样告诉我的：DST是一个松散的团队，自由的联盟，我们有“任务”时，你有权根据自己的时间、精力来选择是否去接受任务，有空上DST论坛发发帖，帮助一下新手就行了。真的很松散，很自由。我加入的时候，正值6.0发布之际，随着而来的，就是大量的主题，插件升级，于是DST在这个时候，推出了一个活动， [D.S.T]风格组免费帮您升级风格。但却引来了一个DZ会员tgwglzc的攻击言论：好象这个贴消失了，说实在的DST就是图有虚名，充门面的骗子。具体你可以看看DST博客的一些真相。

我想，DST这样一个免费的、以热心为基础的组织，从一建立就开始受到各方关注，也从一建立就开始遭到少数不友善的组织、个人的攻击。
包括语言上的诋毁、深入团队内部的蛊惑，以及一些其他方式。
DST招惹了谁？DST何德何能去承受这些不友善的组织和个人不惜金钱、殚精竭虑的攻击？

任何组织都不可能做到尽善尽美，的确，DST也有很多不足。
对于善意的批评、建议，我们都很高兴地接受了。
而对于恶意的攻击、否认、贬低（或者是所谓“激将法”吧），我们实在不敢恭维。我们也是人，我们也会受伤。

其实，就我个人很不明白。
为什么一些我们花费心思、精力去努力制作的作品免费提供下载，说声“谢谢”的人那么少。反而有跳出来恶语相向、诽谤攻击的？
站长们的感恩心哪去了——也许对于DST，还够不上“感恩”的资格。但是多少总有帮助吧？

不盈利，DST本身的动力就很有限。
而DST的工作又是极其损耗精力和时间的。所有的团队成员都有自己的工作、学习、生活。
本身就很不容易，却还要承受这么多的负面影响、攻击。

这年头，想安心地好好做些事情，都这么难么？免费分享的道路已尽？ 我似乎走入了一个误区

现在，Discuz已经发展到了6.1.0，但是，曾经DST成员经常交流，发布作品的Discuz! 官方论坛主题区和插件区已经不见了那火热的DST热情，DST论坛每日的帖数少得可怜。很多成员已经暂时离开DZ，离开了DST。少了很多DST团队的作品。这是一个悲哀，问题出在哪里呢？可能问题很多，但是最大的问题可能是这个团队成立本身的原则之一：非盈利。我的眼睛没有看到DST有任何盈利。所有的风格，插件都是免费发布。所有的成员都是义务劳动。没有任何物资回报，有点只是一些会员的：顶，好，支持，用了，很棒。对DST成员来说这已经够了。但却还有一些会员的诽谤，攻击。面对这些攻击。团队中，也有人突然觉得迷惘起来：他们不说还好，这一说，我倒真的开始思考：我们这么拼死拼活是为了谁？每天24小时，16个小时泡在DZ论坛！

DST发布了很多的作品，风格，插件。比较突出，DZ会员使用最多的应该是：


  1. Discuz! So Easy Server Center

  2. Discuz! Add-ons Center

  3. Discuz! Support Team 论坛默认风格


当然，还有太多太多的作品，太多太多的插件。你可以在DST论坛中找到他们。

如果你用过DST的作品，支持DST的发展，请回复祝福那些默默在用户背后辛苦工作的DST成员。为他们加油，即使他们不再发布作品，不再出现在大家的眼前。

PS：现在的DST论坛已经变成17DST论坛了，也许大家认为它们是相同的，其实不然。DST是17DST的创立者。DST是团队，17DST是平台。DST和17DST是分别独立的。DST是17DST的参与者。
